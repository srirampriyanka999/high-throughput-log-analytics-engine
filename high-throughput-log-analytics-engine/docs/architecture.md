## System Architecture Overview

### Log Sources
Logs are generated by backend services such as authentication, payment, and order services.

### Log Collection
Logs are sent to a centralized log collector using HTTP or agent-based collection.

### Log Storage
Validated logs are stored in centralized storage for querying and analysis.

### Processing & Anomaly Detection
Logs are processed in batches or streams to detect predefined anomalies using rule-based logic.

### Handling Invalid Logs
Logs that fail schema validation are dropped or stored separately for inspection.

### Alerting
When anomalies are detected, alerts are sent to operators via email or messaging platforms.
